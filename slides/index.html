⚠️ สำคัญ: ESP32-CAM ไม่มี USB — ต้องใช้บอร์ด MB (CH340) สำหรับการ Flash Firmware

ส่วนประกอบของ ESP32-CAM
Hardware Tour
  • OV2640 Camera Module — กล้อง 2MP สำหรับถ่ายภาพ
  • ESP32-WROVER SoC — ชิปหลักพร้อม Wi-Fi + Bluetooth
  • GPIO0 — ⚠️ ต่อ GND ขณะ Upload เท่านั้น
  • 5V / GND — จ่ายไฟผ่าน MB Board
  • TX / RX — UART สำหรับ Serial Monitor
  • MicroSD slot — เก็บภาพในเครื่อง (ไม่ใช้ใน Workshop นี้)
⚠️ ขั้นตอน Flash:
1. GPIO0 → GND (jumper)
2. เสียบสาย USB → กด Upload
3. รอ "Writing at 0x..."
4. ถอด jumper → กด RST
MB Board (ESP32-CAM-MB)
มีปุ่ม RST + IO0
ใช้ CH340 USB-to-Serial
Wi-Fi & Networking พื้นฐาน
SSIDชื่อ Wi-Fi Network
Passwordรหัสผ่าน Wi-Fi
IP Addressเช่น 192.168.1.50
Web UI Port:80 (default)
Stream Port:81
Frequency2.4 GHz เท่านั้น
secrets.h
#define WIFI_SSID  "Workshop-WiFi"
#define WIFI_PASS  "password123"

LAB 01 — Overview
ขั้นตอนทั้งหมดของ Lab 01
✅ต่อสาย ESP32-CAM เข้า MB Board
✅ติดตั้ง Arduino IDE + ESP32 Board Package
✅แก้ไขไฟล์ secrets.h ใส่ Wi-Fi ของห้อง
✅Upload (Flash) Firmware ไปยัง ESP32-CAM
✅ทดสอบ Live Stream และ /capture endpoint
LAB 01
Step 1 — ต่อสาย ESP32-CAM
1
เสียบ ESP32-CAM ลงบน MB Board (ให้ช่องกล้องหันออก)
2
ต่อสาย Micro-USB จาก MB Board ไปยัง Notebook
3
กดปุ่ม IO0 ค้างไว้ (หรือใส่ jumper GPIO0→GND) ก่อน Upload
⚠️ GPIO0 Flash Mode:
• Flash: GPIO0 → GND (jumper)
• Run: ถอด jumper → กด RST

ถ้าลืมถอด jumper: ESP32 จะไม่ boot!
LAB 01
Step 2 — Arduino IDE Setup
1
File → Preferences → ใส่ Additional boards manager URL:
https://dl.espressif.com/dl/package_esp32_index.json
2
Tools → Board → Boards Manager → ค้นหา "esp32" → Install
3
Tools → Board → เลือก AI Thinker ESP32-CAM
4
Tools → Port → เลือก COMx ที่มี "(CH340)"
เปิด Sketch: CameraWebServer/CameraWebServer.ino
LAB 01
Step 3 — แก้ไข secrets.h
secrets.h
#define WIFI_SSID  "ชื่อ Wi-Fi ของห้อง"
#define WIFI_PASS  "รหัสผ่าน Wi-Fi"
⚠️ อย่า commit secrets.h!
ไฟล์นี้อยู่ใน .gitignore แล้ว
ไม่มีใน GitHub repo
ใช้ secrets.example.h เป็น template
copy → rename → แก้ค่า Wi-Fi
⚠️ Wi-Fi 2.4 GHz เท่านั้น — ถ้าไม่แน่ใจ ให้ถามผู้สอน
LAB 01
Step 4 — Flash Firmware
1
ตรวจสอบว่า GPIO0 ต่อ GND (jumper ใส่ไว้)
2
กด Upload (ลูกศรชี้ขวา) ใน Arduino IDE — รอ "Connecting..."
3
รอจนเห็น "Writing at 0x00001000..." — กำลัง flash
4
เห็น "Hash of data verified" → Upload สำเร็จ!
5
ถอด jumper GPIO0 → กดปุ่ม RST บน MB Board
ถ้า Connecting... ค้าง: กดปุ่ม IO0 ค้างไว้ระหว่าง Upload
LAB 01
Step 5 — หา IP และทดสอบ
1
เปิด Serial Monitor (Tools → Serial Monitor) ตั้ง Baud Rate: 115200
2
กด RST → รอ 5 วินาที → จะเห็น IP Address เช่น 192.168.1.50
3
เปิด Browser → พิมพ์ http://192.168.x.x → เห็น Web UI
4
ทดสอบ http://192.168.x.x/capture → ได้ภาพ JPEG
5
ทดสอบ http://192.168.x.x:81/stream → เห็น Live Video
LAB 01 — Expected Result
ผลลัพธ์ที่ควรได้
✅ESP32-CAM เชื่อมต่อ Wi-Fi สำเร็จ — ไฟ LED กะพริบ
✅Serial Monitor แสดง IP Address เช่น 192.168.1.50
✅Browser เปิด http://<IP> แสดงหน้า Web UI
✅/capture endpoint ส่ง JPEG กลับมา
✅:81/stream แสดง Live Video Stream
จด IP Address ไว้! จะใช้ใน Lab 02 และ 03

Template

[หัวข้อ 1]: [ค่า]
[หัวข้อ 2]: [ค่า]

• ยิ่ง Prompt ชัดเจน ผลดียิ่งขึ้น

• ทดสอบซ้ำ → ปรับปรุง

🗑️ Waste Classification — จำแนกขยะ
Prompt
คุณคือผู้เชี่ยวชาญการจัดการขยะ วิเคราะห์ขยะที่เห็นในภาพ จำแนกแต่ละชิ้นเป็น: - ♻️ รีไซเคิล (กระดาษ/พลาสติก/แก้ว) - 🌿 อินทรีย์ (เศษอาหาร/ใบไม้) - ☢️ อันตราย (แบตเตอรี่/เคมี) ตอบเป็น bullet point ภาษาไทย สรุปคำแนะนำ 1 ประโยค
Expected Output:
• กล่องกระดาษ: ♻️ รีไซเคิล
• ขวดน้ำ: ♻️ รีไซเคิล
• เปลือกผลไม้: 🌿 อินทรีย์

สรุป: แยกทิ้งให้ถูกถัง...
ประยุกต์ใช้: ถังขยะอัจฉริยะ
🌿 Plant Health Analysis — สุขภาพพืช
Prompt
คุณคือนักพฤกษศาสตร์ผู้เชี่ยวชาญ วิเคราะห์สุขภาพพืชในภาพ ตอบในรูปแบบ: ชนิดพืช: [ชื่อ] สุขภาพ: [1-10] สี: [สีใบ/ลักษณะ] ปัญหา: [โรค/ศัตรู ถ้ามี] คำแนะนำ: [วิธีดูแล] ตอบเป็นภาษาไทย
Expected Output:
ชนิดพืช: ต้นมะนาว
สุขภาพ: 7/10
สี: เขียวสด มีจุดเหลืองบ้าง
ปัญหา: อาจขาดน้ำเล็กน้อย
คำแนะนำ: รดน้ำเช้าเย็น...
ประยุกต์ใช้: ระบบดูแลสวนอัตโนมัติ
🔢 Object Counting — นับวัตถุ
Prompt
นับวัตถุทุกชิ้นในภาพนี้ แยกตามประเภท ตอบในรูปแบบ: [ประเภทวัตถุ]: [จำนวน] ชิ้น รวมทั้งหมด: [X] ชิ้น ประเภทที่มากที่สุด: [ชื่อ] ตอบเป็นภาษาไทย
Expected Output:
ปากกา: 5 ชิ้น
หนังสือ: 3 เล่ม
แก้วน้ำ: 1 ใบ
โทรศัพท์: 1 เครื่อง

รวม: 10 ชิ้น
มากที่สุด: ปากกา
ประยุกต์ใช้: ระบบนับสินค้าคลัง
🖥️ Desk Cleanliness — ความเป็นระเบียบ
Prompt
คุณคือผู้ตรวจสอบพื้นที่ทำงาน ประเมินความเป็นระเบียบของโต๊ะในภาพ ตอบในรูปแบบ: คะแนนความเป็นระเบียบ: [1-10] จุดเด่น: [สิ่งที่ดี] จุดที่ควรปรับปรุง: [bullet] คำแนะนำ: [วิธีจัดระเบียบ] ตอบเป็นภาษาไทย
Expected Output:
คะแนน: 6/10
จุดเด่น: จอคอมจัดวางดี
ปรับปรุง:
• กระดาษกองสูงเกินไป
• สายไฟพันกัน
คำแนะนำ: จัดหมวดเอกสาร...
ประยุกต์ใช้: ระบบ 5ส อัตโนมัติ
🔧 Equipment Detection — ตรวจอุปกรณ์
Prompt
คุณคือช่างเทคนิคผู้เชี่ยวชาญ ตรวจสอบอุปกรณ์ที่เห็นในภาพ สำหรับอุปกรณ์แต่ละชิ้น ระบุ: - ชื่ออุปกรณ์ - สถานะ: ✅ปกติ / ⚠️ต้องตรวจสอบ / ❌ผิดปกติ - หมายเหตุ (ถ้ามี) ตอบเป็นภาษาไทย
Expected Output:
• ESP32-CAM: ✅ ปกติ
• สาย USB: ✅ เชื่อมต่อแล้ว
• Breadboard: ⚠️ มีสายหลวม
• LED: ✅ ติดสว่าง

พบ 1 จุดที่ควรตรวจสอบ
ประยุกต์ใช้: ระบบตรวจสอบไลน์ผลิต
เส้นทางสายอาชีพในอนาคต
