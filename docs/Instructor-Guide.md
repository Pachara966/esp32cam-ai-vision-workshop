## ก่อนวัน Workshop / Pre-Workshop Checklist

### 1 สัปดาห์ก่อน

- [ ] ทดสอบ flash firmware ลง ESP32-CAM ให้ครบทุก unit (หรืออย่างน้อย 1 ชุดต่อกลุ่ม)
- [ ] ติดตั้ง Arduino IDE + ESP32 Board Package บน laptop ที่ใช้สอน
- [ ] ทดสอบรัน backend ครบ end-to-end: mock → gemini → claude
- [ ] ตรวจสอบ Wi-Fi ในห้อง: รองรับ 2.4 GHz, ไม่มี client isolation
- [ ] เตรียม API key สำรอง (Gemini) อย่างน้อย 2 key
- [ ] สั่งซื้อ/ตรวจสอบ ESP32-CAM + MB board ให้ครบ (+ สำรอง 2-3 ชุด)
- [ ] ติดตั้งไดรเวอร์ CH340 บน laptop นักเรียนล่วงหน้า (ถ้าทำได้)

- [ ] ทดสอบ Wi-Fi สัญญาณในห้องจริง

## Common Problems & Quick Fixes

| ปัญหา | สาเหตุที่พบบ่อย | วิธีแก้ด่วน |
|-------|---------------|------------|
| Upload ไม่ได้ | ไม่กด IO0 ก่อน upload | กด IO0 + RST แล้ว upload ใหม่ |
| ไม่เห็น COM Port | ไม่มีไดรเวอร์ CH340 | ติดตั้ง CH340 driver แล้ว restart |
| Wi-Fi ไม่เชื่อม | SSID/Pass ผิด หรือ 5GHz | เช็ค secrets.h, ใช้ 2.4GHz เท่านั้น |
| `Camera init failed` | กล้องหลวม หรือไฟไม่พอ | เสียบใหม่ให้แน่น หรือเปลี่ยนสาย USB |
| pip install ช้า/fail | อินเทอร์เน็ตช้า | แชร์ .whl cache หรือใช้ mobile hotspot |
| API 429 quota | หลายคนใช้ key เดียวกัน | แจก key สำรอง หรือสลับเป็น mock |
| `Internal Server Error` | API key ผิด หรือ model ไม่พร้อม | เช็ค .env, ลอง VISION_PROVIDER=mock |

ดูรายละเอียดเพิ่มเติมใน [`lab/Troubleshooting-Guide.md`](../lab/Troubleshooting-Guide.md)

## Tips สำหรับผู้สอน

**การอธิบาย API ให้เข้าใจง่าย:**
> "API คือ 'ช่องทาง' ที่โปรแกรมคุยกัน เหมือน drive-through ของร้านอาหาร — เราสั่ง (request) ผ่านหน้าต่าง แล้วได้รับอาหาร (response) กลับมา"

**การอธิบาย Prompt Engineering:**
> "Prompt คือ 'วิธีพูดกับ AI' ถ้าพูดกว้างๆ AI จะตอบกว้างๆ ถ้าพูดเจาะจง AI จะตอบตรงกว่า ลองเปรียบกับการสั่งอาหาร: 'อยากกินข้าว' vs 'ขอข้าวผัดหมู ไม่เผ็ด ไข่ดาว'"

**เมื่อกลุ่มไหนทำเสร็จเร็ว:**
- ลองเปลี่ยน model (claude แทน gemini)
- ลอง prompt เป็นภาษาอังกฤษแล้วเปรียบเทียบผล
- ลองชี้กล้องไปที่สิ่งที่ AI น่าจะวิเคราะห์ยาก

**เมื่อกลุ่มไหนติดขัด:**
- ถามก่อนว่า "ตอนนี้เห็น error อะไร?" ไม่ใช่แก้ให้ทันที
- ให้นักเรียนอ่าน error message ออกเสียงดังๆ — มักช่วยให้เห็นปัญหาเอง
